How to upload files to your department intranet

Policies, guides, documents — if your colleagues keep asking you for the same thing, upload it once and let the intranet do the work for you.

1

Can you even edit the page?

Let's find out first

Go to your department's SharePoint site. Look at the top right corner of the page. If you see an ✏ Edit button — congratulations, you are the chosen one. Skip to step 2.

🙃 If there is no Edit button and you cannot see Site contents in the settings menu — you do not have edit access. This is not a bug. This is not your computer. You simply have not been given permission yet.

Contact your HOD or the Digital team and request Member or Contributor access to the site. We promise we will not judge you for not knowing.

How to check: Click the ⚙ gear icon at the top right → if you can see "Site contents" in the menu, you have some level of access. If that option is missing entirely, you definitely need to request access first.

2

Open Site contents and find the Document Library

This is where the files live

Click the ⚙ gear icon → Site contents. You will see a list of all the lists, libraries, and apps on your site. Look for the one that has the most items — that is your Document Library. It is usually called Documents or named after your department.

Click on the Document Library to open it. This is where you will upload your file.

3

Find the right folder — or create one

Please do not just dump it at the root level

Inside the Document Library, look for a folder that matches what you are uploading. We use these standard folders so that anyone who lands on the page is not playing a guessing game:

📁 Policies

Company-wide rules, HR policies, compliance documents, codes of conduct.

📁 Guides

How-to documents, step-by-step instructions, FAQs, user manuals.

📁 Templates

Reusable document templates, letter formats, report structures.

📁 Reports

Monthly, quarterly, or annual reports shared across the organisation.

If the folder you need does not exist yet, here is how to create it:

Upload
📁 Folder ← click this
📄 Word document
📊 Excel workbook

Click + New → Folder → name it correctly (see naming rules below) → click Create.

4

Name your file properly before uploading

This part is important — everybody will see this name

Before you upload anything, make sure the file name is clean and professional. The name you upload is the name everyone sees.

🫠 Yes, you can upload a file called "FINAL_v3_ACTUAL_FINAL_useThis.docx" or "document1.pdf". SharePoint will not stop you. Your colleagues, however, will silently judge you. Forever.

Follow this naming convention:

Guides
Guide_HowToRaiseAnExpenseClaim.pdf
guide expenses thing final.pdf
Policies
Policy_LeaveAndAbsence_2026.pdf
HR policy new version.docx
Templates
Template_MeetingMinutes.docx
template1.docx
Reports
Report_Q1Sales_2026.xlsx
salesreport.xlsx

Rule of thumb: If a stranger opened the file and saw only its name, would they know exactly what it is? If yes — good name. If no — rename it.

5

Upload the file into the correct folder

Almost there!

Open the folder you want to upload into (e.g. Guides), then do one of the following:

⬆ Upload
+ New
📄 Files ← most common
📁 Folder

Click Upload → Files → select your file from your computer → click Open. Or simply drag and drop the file directly into the folder — that works too.

Double check you are inside the right folder before uploading. If you upload at the root level of the library instead of inside a folder, your file will be floating there with no context and someone will have to move it. That someone may be you.

6

You're done — now go tell your team

The file is live. Don't leave them guessing.

Your file is now on the intranet and visible to everyone with access to the site. SharePoint does not send an automatic notification, so drop a quick message on Teams to let your colleagues know it's there.

Pro tip: Copy the direct link to the file (right-click → Copy link) and paste it into your Teams message so people can jump straight to it without hunting through folders.
